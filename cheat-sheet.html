หน้าแรก แผนการเรียนรู้ เริ่มต้น ความสามารถ เมนูอธิบาย Cheat Sheet อ้างอิง

Quick Commands

Cheat Sheet สั่งงานเร็ว

รวมคำสั่งสำเร็จรูปที่คัดลอกไปใช้ได้ทันที แยกตามงานเพื่อหาง่าย

ใช้ยังไง

คัดลอกประโยคที่ใกล้เคียงงานของคุณ แล้วเติมรายละเอียดเฉพาะโปรเจกต์

คำสั่งพื้นฐาน

เริ่มงานใหม่หรือสั่งงานทั่วไป

เริ่มงานใหม่

  • “สร้างเธรดใหม่สำหรับงาน X และตั้งชื่อให้จำง่าย”
  • “เปิดงานใหม่เป็น Worktree สำหรับฟีเจอร์ Y”
  • “สรุปเป้าหมายงานนี้ให้ชัดก่อนเริ่ม”

สรุปงาน

  • “สรุปผลลัพธ์เป็น bullet list พร้อมไฟล์ที่เปลี่ยน”
  • “ทำสรุปสำหรับทีมแบบ 5 ข้อ”
  • “อธิบายให้มือใหม่เข้าใจใน 3 ย่อหน้า”

Prompt สูตรสำเร็จ

“เป้าหมาย + บริบท + ข้อจำกัด + รูปแบบผลลัพธ์”

ตัวอย่าง

เป้าหมาย: แก้บั๊ก login
บริบท: ไฟล์ src/auth.ts
ข้อจำกัด: ห้ามเปลี่ยน API
รูปแบบผลลัพธ์: สรุปเป็น bullet list

คำสั่งสำหรับงานโค้ด

ใช้กับการแก้โค้ด รีแฟกเตอร์ และทดสอบ

แก้บั๊ก

  • “หาสาเหตุบั๊กนี้และเสนอวิธีแก้ 2 ทาง”
  • “แก้ให้หายและสรุปสิ่งที่เปลี่ยน”
  • “เพิ่มเทสต์กันบั๊กนี้ไม่ให้กลับมาอีก”

รีแฟกเตอร์

  • “รีแฟกเตอร์ให้อ่านง่ายขึ้น โดยไม่เปลี่ยน behavior”
  • “แยกฟังก์ชันใหญ่ให้เป็นฟังก์ชันย่อย”
  • “เพิ่มคอมเมนต์เฉพาะส่วนที่ซับซ้อน”

ทดสอบ

  • “รันเทสต์ทั้งหมด แล้วสรุปผลให้ที”
  • “แนะนำ edge case ที่ยังไม่ถูกเทสต์”
  • “เพิ่มเทสต์ให้ครอบคลุมเส้นทางสำคัญ”

คำสั่งสำหรับงานทีม

เหมาะกับการรีวิว ส่งงาน หรือสรุปให้ทีม

รีวิวโค้ด

  • “ช่วยดู diff และชี้จุดเสี่ยงที่อาจพังได้”
  • “ให้ feedback แบบมืออาชีพและเสนอทางเลือก”
  • “สรุปประเด็นหลักให้ทีมอ่านใน 1 นาที”

เตรียม PR

  • “สร้าง commit message ให้สั้น กระชับ”
  • “สรุป PR ในรูปแบบ template ที่ทีมใช้”
  • “แนะนำ checklist ก่อน merge”

เอกสาร/สรุป

  • “สรุปเอกสารนี้เป็น 5 ประเด็น”
  • “ทำ checklist ให้ทีมทำตามทีละขั้น”
  • “แปลงเป็นคู่มือภาษาไทยพร้อมตัวอย่าง”
